# Архитектура операционных систем

**Архитектура операционных систем (ОС)** — это структурная и функциональная организация ОС на основе совокупности программных модулей. Единой унифицированной архитектуры ОС не существует, но известны универсальные подходы к структурированию. Некоторые типы архитектур: монолитная, микроядерная и гибридная.

## Монолитная структура ОС

Классическая архитектура, при которой все компоненты ОС (управление процессами, памятью, файловыми системами, драйверы устройств) работают в едином адресном пространстве ядра в привилегированном режиме.

1. Преимущества: Высокая производительность за счет отсутствия накладных расходов на межпроцессное взаимодействие (IPC); простота взаимодействия между модулями.

2. Недостатки: Низкая надежность (ошибка в любом драйвере может привести к краху всей системы); сложность поддержки и модификации.

3. Назначение: Универсальные серверы, компьютеры, где важна скорость.

4. Примеры: Linux, традиционные UNIX, Windows (в ранних версиях).

5. Плюсы/Минусы:

    - :material-check-outline:{ .lg .middle style="color: #00FF00" } Высокая производительность.

    - :material-close-outline:{ .lg .middle style="color: #ff0000" } Низкая надежность (падение драйвера = падение всей системы).
  
## Микроядерная структура ОС

Подход, минимизирующий функционал ядра. В привилегированном режиме остается лишь самый необходимый набор: низкоуровневое управление адресными пространствами, диспетчеризация потоков и межпроцессное взаимодействие (IPC). Все остальные компоненты (драйверы, файловые системы, сетевые стеки) выполняются в пользовательском пространстве как обычные сервисы.

1. Преимущества: Высокая отказоустойчивость (сервис можно перезапустить без остановки системы); строгая модульность и изоляция; безопасность.

2. Недостатки: Снижение производительности из-за частого переключения контекста и накладных расходов на IPC.

3. Назначение: Критическая инфраструктура (медицина, авиация), системы реального времени.

4. Примеры: QNX, L4, Symbian OS, GNU Hurd.

5. Плюсы/Минусы:

    - :material-check-outline:{ .lg .middle style="color: #00FF00" } Высокая отказоустойчивость (драйвер можно перезапустить на лету).

    - :material-close-outline:{ .lg .middle style="color: #ff0000" } Медленнее работает из-за постоянных переключений между режимами.

## Гибридная структура ОС

Компромиссная архитектура, объединяющая производительность монолитного ядра с элементами модульности микроядра. В ядро включаются критические компоненты, но многие подсистемы (особенно драйверы) выносятся в пользовательское пространство или работают в изолированных режимах.

1. Особенности: Использование модульной загрузки (kernel modules) без необходимости перекомпиляции ядра; поддержка драйверов в режиме пользователя.

2. Назначение: Современные настольные и серверные ОС.

3. Примеры: Microsoft Windows (NT) , macOS (XNU).

4. Плюсы/Минусы:

    - :material-check-outline:{ .lg .middle style="color: #00FF00" } Баланс скорости и стабильности.

    - :material-check-outline:{ .lg .middle style="color: #00FF00" } Возможность загружать модули без перезагрузки.

---

Архитектура ОС представляет собой сложный баланс между производительностью, безопасностью и модульностью. Современные ОС все чаще отходят от строгих монолитных моделей в сторону гибридных решений с возможностью изоляции критических компонентов, что позволяет удовлетворять требованиям высоконагруженных серверных систем, встраиваемой техники и критической инфраструктуры.
